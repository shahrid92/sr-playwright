## Commands

```bash
# Run all tests
npx playwright test

# Run specific test file
npx playwright test tests/e2e/admin-management.smoke.spec.ts

# Run tests with a specific tag (e.g., @smoke, @TC001)
npx playwright test --grep @smoke
npx playwright test --grep @TC001

# Run tests for a specific project (chromium, API, setup, chromium-setup)
npx playwright test --project=chromium

# Run a single shard across multiple workers (used by the queue system)
npx playwright test --shard=1/4 --workers=4 --project=chromium --grep=@smoke

# Run tests in headed mode (visible browser)
npx playwright test --headed

# Run tests with UI mode
npx playwright test --ui

# Generate BDD step definitions from feature files
npm run bddgen

# Regenerate barrel exports from page objects
npm run barrel

# Generate & serve Allure report
npm run alluregen

# List tests matching a grep pattern
npx playwright test --grep @smoke --list --project=chromium

# Auth setup (generates storage state)
npx playwright test tests/e2e/auth.setup.ts --project=setup

# Add tests to BullMQ queue (distributed execution)
node queue/bull-mq/producer/producer.js <shard-count>
```

## Architecture

This is a **Playwright E2E testing framework** targeting [OrangeHRM](https://opensource-demo.orangehrmlive.com), a demo HR management app. Tests cover the Admin module (user management, job titles, qualifications, corporate branding, organization, navigation).

### Test Structure

- **`tests/e2e/`** — Standard Playwright spec files using the Page Object Model. `auth.setup.ts` handles authentication via storage state. `admin-management.smoke.spec.ts` is the main smoke test suite (9 test cases across Admin features).

- **`tests/pages/`** — Page Object Model classes (`login.ts`, `dashboard.ts`, `admin.user-management.ts`). Barrel-exported via `tests/index.ts` (auto-generated by `barrelsby`).

- **`tests/bdd/`** — BDD tests using `playwright-bdd` (v8). Gherkin feature files in `features/`, step definitions in `steps/`, and a separate fixture set in `fixtures/testFixtures.ts`. BDD uses its own test base imported from `playwright-bdd`, distinct from the standard `@playwright/test` base.

- **`fixtures/`** — Shared test fixtures (initially auto-login via route intercept, plus `LoginPage`/`DashboardPage` injection). Barrel-exported and importable as `@fixtures`.

### Authentication

Uses Playwright storage state (`tests/playwright/.auth/user.json`). The `setup` project runs `auth.setup.ts` which logs in once with "Admin"/"admin123", saves cookies/localStorage, then the `chromium-setup` project reuses that state via `storageState`. The fixture file also performs auto-login in `loginPage` fixture setup with route interception for debugging.

### CI/CD & Distributed Execution

Tests run via **Helm on Kubernetes** with a self-hosted GitHub runner:

1. **Helm chart** (`infra/helm/helm-playwright/`) deploys a Kubernetes Job that clones the repo, installs deps, runs `npx playwright test` with configurable tags/workers/project.

2. **BullMQ queue system** (`queue/bull-mq/`) enables sharded execution:
   - **Producer** pushes shard jobs to a Redis queue (`playwright-tests`).
   - **Consumer/worker** (`consumer.js`) picks up jobs and runs `npx playwright test --shard=N/M --grep=@smoke`.

3. **KEDA ScaledJob** (`infra/keda/`) auto-scales workers based on Redis queue length (max 20 replicas).

4. **Docker image** (`Dockerfile.playwright_consumer`) packages the consumer with Playwright v1.59.1.

### Reporting

Three reporters configured simultaneously:
- **List** (terminal output)
- **HTML** (saved to `tests/test-results/html-report/`)
- **Allure** (results in `tests/test-results/allure-results/`, generated via `npm run alluregen`)
- **JSON** (saved to `tests/test-results/test-results.json`)

### Playwright Config

Multi-project setup in `playwright.config.ts`:
- `setup` — Authenticates and saves storage state
- `API` — API-only tests
- `chromium-setup` — Chromium tests using pre-authenticated state (depends on `setup`)
- `chromium` — Standalone Chromium tests (no auth dependency)

Base URL: `https://opensource-demo.orangehrmlive.com/web/index.php/auth/login`

### VSCode Integration

VSCode settings enable the Playwright MCP server (`@playwright/mcp`) for AI-assisted browsing.

## Key Patterns

- Tests use tags (`@smoke`, `@TC001`) for filtering and annotations from `@faker-js/faker` for test data generation.
- Page objects accept `page: Page` in constructors and expose typed `Locator` properties.
- The `adminPage` class uses XPath and CSS selectors targeting OrangeHRM's `oxd-*` CSS framework.
- The CI workflow tracks job completion via `kubectl` polling (36 retries × 10s interval).
